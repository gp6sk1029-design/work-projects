# メール返信ツール

## 概要
受信メールに対してビジネスメールとして適切な返信文を日本語で生成する。
**手動モード**（その場で返信文を作成）と**自動モード**（Thunderbird常駐・下書き自動保存）の2つの使い方がある。

---

## 手動モード（その場で返信文を作成）

### 使い方
以下の情報を貼り付けるだけでOK。

```
【件名】（例：お見積りのご依頼について）
【送信者】（例：株式会社〇〇 田中様）
【本文】
（メール本文をそのまま貼り付け）

【過去のやり取り】（任意）
（関連する過去メールがあれば貼り付け）

【返信の意図】（任意）
（例：承諾する／断る／確認する／お礼を伝える）
```

### 処理手順

1. **文脈の読み取り**
   - 件名・送信者・本文から要件を把握する
   - 過去のやり取りがあれば流れを確認する
   - 送信者との関係性（初対面／継続取引など）を推定する

2. **返信方針の確認**
   - 返信の意図が未指定の場合、適切な対応を提案してユーザーに確認する
   - 断る・遅延するなど慎重な対応が必要な場合は一言添えて確認する

3. **返信文の生成と出力**
   - 確認用として出力し「この文面でよろしければコピーしてご使用ください」と案内する
   - 修正があれば再生成する
   - **自動送信は絶対に行わない**

### 文体ルール

| 状況 | 文体の方向性 |
|------|------------|
| 初めての相手 | 丁寧・堅め |
| 継続取引の相手 | 丁寧・やや柔らか |
| 社内向け | 敬語を保ちつつ簡潔 |
| お断り・遅延 | 丁寧・配慮を込める |
| お礼・承諾 | 明るく・簡潔 |

## 自動モード（Thunderbird常駐・下書き自動保存）

### 仕組み
`auto_draft.py` を起動するとバックグラウンドで以下を60秒ごとに繰り返す。

```
Thunderbird起動中？ → いいえ → 待機
  ↓ はい
受信から5分以内のメール？ → いいえ → スキップ
  ↓ はい
返信が必要？（迷惑・メルマガ除外） → いいえ → スキップ
  ↓ はい
過去のやり取りを取得
  ↓
Gemini APIで返信文生成
  ↓
Thunderbirdの下書きフォルダに保存
```

### 起動方法
```
run.bat をダブルクリック
```

### 設定ファイル（.env）
```
MY_EMAIL=sy-kouda@ime-group.co.jp
MY_NAME=幸田
GEMINI_API_KEY=（GeminiのAPIキー）
```

### 注意事項
- 下書き保存のみ（自動送信はしない）
- 送信前に必ず内容を確認・修正してから送ること
- 金額・納期・契約に関わる内容は特に慎重に確認すること
